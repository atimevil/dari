## 1. 문제 정의

중국·베트남 등 외국인 유학생은 **D-2 비자 연장, 외국인등록, 시간제취업허가** 같은 복잡하고 고위험인 출입국 행정을, **모국어 정보 없이** 하이코리아·행정실·커뮤니티를 헤매며 처리한다. 실수 시 **불법체류 등 치명적 위험**에 노출된다.

**근거 (시장조사):**
- 정부 'Study Korea 30만명' 유학생 유치 정책 → 유학생 급증, 특히 지방대 생존 전략과 직결
- 기존 정보 경로(하이코리아·행정실·온라인 커뮤니티)는 **한국어 중심·분산·비개인화**
- 한국 대학에서 생성형 AI 행정챗봇 도입은 시작됐으나(대구한의대·경희대·서울시립대·중앙대 등) **각 대학이 자체적으로, 따로따로** 구축 중이며 통합 제품은 없음

## 2. 타겟 사용자

- **1차 (MVP)**: 국내 중국·베트남 유학생 (최대 규모, 초기 트랙션 유리)
- **확장**: 영어권·중앙아시아(우즈벡·몽골) 등 전 언어권 유학생
- **B2B 고객 (사업화)**: 대학 국제처 — 유학생 유치·중도탈락 방지·행정부담 경감이 도입 동기

## 3. 경쟁 검증 — 왜 '백지'인가

검색으로 확인한 결과:

| 인접 영역 | 현황 | 비고 |
|---|---|---|
| 단일대학 행정챗봇 | 대구한의대 'AI기린' 등 다국어 챗봇 | 단일 대학 한정, 통합·개인화 부재 |
| 일반 학생앱 | 에브리타임(837만), 대학백과, 노크 | 내국인·한국어 중심 |
| 노트/시험대비 AI | 다글로, 클로바노트, AI레포토 | 학습용, 비자·체류 무관 |
| 취업/멘토링 | 코멘토, 잇다 | 한국어·취업 중심 |

→ **통합·개인화·다국어·출처기반 비자/체류 AI는 한국 시장 부재.** 한국 출입국법·행정의 로컬라이징 자체가 진입장벽이자 해자.

## 4. 킬러 기능 — 개인화 다국어 비자 Q&A

**UX 흐름:**
1. **온보딩**: 체류자격(D-2 세부)·입국일·비자 만료일·외국인등록 여부·학교·학기·아르바이트 여부 입력 (외국인등록증/비자 OCR 자동입력은 향후)
2. **질문**: 모국어로 자유질문 — 예) "겨울방학에 알바 시간 늘릴 수 있어요?"
3. **처리**: 개인 프로필 + 제도/학교 컨텍스트 + 공식문서(RAG) 결합
4. **답변**: 모국어 답변 + ✅절차 체크리스트 + 📄필요서류 + ⏰기한 + 🔗공식 출처 인용
5. **안전장치**: 고위험·불확실 사안 → 출입국·국제처(·향후 행정사) 연결 에스컬레이션

**일반 ChatGPT 대비 차별 4요소:**
1. 내 체류상황 기억 (개인화)
2. 공식 출처 grounding (환각·오안내 차단)
3. 모국어 응답
4. 학교별 행정 절차 반영

## 5. 시스템 아키텍처

- **입력층**: 다국어 질문 파싱(의도·엔티티 추출) + 프로필 컨텍스트 주입
- **지식층 (RAG)**: 출입국·하이코리아 공식 안내 + 비자별 매뉴얼 (+향후 제휴 대학 국제처 자료 = 독점 데이터)
- **생성층**: LLM이 출처를 근거로 모국어 답변 + 체크리스트 생성, "모르면 모른다" 강제
- **신뢰 게이트**: 답변 위험도 분류 → 고위험/저신뢰 시 자동 에스컬레이션
- **(운영) 업데이트 파이프라인**: 출입국 규정 변경 모니터링 → 지식 갱신 → 검수

> **해커톤 단순화**: 핵심 공식문서 ~15개만 큐레이션 → **벡터DB 없이 카테고리별 컨텍스트 주입**으로 충분. (코퍼스가 작아 이 방식이 더 빠르고 안정적)

## 6. 데이터 & 정확성·신뢰 모델 *(제품의 생사를 가르는 부분)*

출입국 오안내는 학생에게 치명적이므로:
- **포지셔닝**: "법률자문이 아니라 공식 정보 안내 + 절차 가이드"로 명확히 한정
- 모든 답변에 **공식 출처 인용 의무화** — 근거 없으면 답변 보류
- 고위험 사안은 **에스컬레이션**, 면책 고지
- **피드백 루프**: 오답 신고 → 검수 반영

## 7. MVP 범위 (해커톤 — 하루 안에 빌드)

| 포함 (IN) | 제외 / 향후 (OUT) |
|---|---|
| 웹앱 + 프로필 입력 폼(비자종류·만료일·등록여부·알바) | 모바일 네이티브 앱 |
| 비자 Q&A (LLM + 큐레이션 공식문서) | OCR 자동입력 (시간 남으면) |
| 중국어·베트남어·한국어 답변 | 전 언어 지원 |
| 출처 인용 + 절차 체크리스트 | 실시간 하이코리아 연동 |
| 비자 만료 D-day 계산·경고 | 행정사 실제 연계(메시지만) |
| 고위험 에스컬레이션 안내 | 대학 B2B 대시보드 |

## 8. 기술 스택 (속도 우선)

- **프론트**: React/Next.js 또는 Streamlit(더 빠른 프로토타이핑)
- **LLM**: Claude API — 다국어 품질 위해 **Sonnet 4.6** 권장
- **지식베이스**: 큐레이션 문서 컨텍스트 주입(해커톤), 향후 벡터DB
- **개인화**: 폼 입력 → 프롬프트 주입
- **마감 계산**: 입력 날짜 기반 단순 날짜연산

## 9. 성공 지표

**해커톤:**
- 동작하는 데모 완성
- 2개 언어(중국어·베트남어) 시연
- 출처 기반 답변(환각 아님) 시연
- 비자 만료 D-day 경고 시연
- 고위험 에스컬레이션 시연

**사업화 (향후):**
- 파일럿 대학 1~2곳 계약
- 유학생 MAU / 질문 수
- 답변 정답률(행정사 검수 기준)
- 에스컬레이션율
- 국제처 문의 감소율(대학 가치 증명) / 재계약 의향

## 10. 리스크 & 대응

| 리스크 | 대응 |
|---|---|
| 법적 정확성·책임 | grounding + 검수 + 면책 + 에스컬레이션, "자문 아님" 포지셔닝 |
| 규정 변경 추적 | 공식출처 모니터링 + 갱신 파이프라인 |
| 환각/오안내 | RAG grounding, 불확실 회피, 피드백 루프 |
| 데모 데이터 부정확 | 큐레이션 문서 출처 검증(공식자료만 사용) |
| (사업) 대학 영업 사이클 김 | 파일럿·대학혁신지원사업 예산 활용, 빠른 레퍼런스 |

## 11. 향후 확장 로드맵 (post-hackathon)

- **기능 확장**: 비자(진입점) → 생활정착(통신·은행·병원·주거) → 한국어·학사 → 취업까지 유학생 라이프사이클 올인원 에이전트
- **사람 결합 해자**: 유학생 전용 **선배-후배 near-peer 멘토링**(같은 출신국·같은 학교). AI(정보) + 사람(경험·신뢰)가 서로의 약점을 메우는 구조 — 기존 멘토링 플랫폼의 'one-and-done' 약점도 AI가 완화
- **수익모델**: 대학 B2B SaaS + 생활서비스 제휴(통신·은행·보험) + 정부 유학생 유치사업 + 행정사 1:1 프리미엄
