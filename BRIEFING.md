# StayWise (체류메이트) — 해커톤 브리핑

> 팀 정렬용 + 발표(피칭) 대본. 빠르게 읽고 바로 빌드/발표에 쓰는 문서.
> 상세 설계는 `SPEC.md` 참고. (작성일 2026-06-22)

---

## 0. 30초 엘리베이터 피치

> "한국 유학생은 비자·체류 행정을 **한국어로만, 흩어진 정보로** 헤매다 실수로 **불법체류 위험**까지 집니다. **StayWise**는 학생의 체류상황을 아는 AI가, **모국어로, 공식 근거와 함께 24시간** 안내합니다. 시중엔 한국어·정보제공 앱뿐, 개인화·다국어·출처기반 비자 AI는 **없습니다**."

## 1. 문제 (Why now)

- 정부 'Study Korea **30만명**' 유치 정책 → 유학생 급증, 지방대 생존 전략과 직결
- 비자 연장·외국인등록·시간제취업허가는 **복잡·고위험** → 실수 시 불법체류
- 기존 정보(하이코리아·행정실)는 **한국어·분산·비개인화** → 유학생 정보 비대칭 극심

## 2. 솔루션 (What)

내 체류상황을 아는 AI 비자 도우미.
**개인화 + 다국어 + 공식 출처 기반 + 비자 특화** = 일반 ChatGPT가 못 하는 4가지를 결합.

## 3. 데모 대본 (약 3분)

**Step 1 — 프로필 셋업** *(미리 입력해두고 시작)*
> "베트남 유학생 Linh, 비자 D-2, 외국인등록 완료, 만료일 곧 도래(D-43), 현재 주 20시간 알바."

**Step 2 — 모국어 질문**
> 베트남어로 입력: *"겨울방학에 알바 시간을 더 늘릴 수 있나요?"*

**Step 3 — 답변 (핵심 장면)**
> ✅ 베트남어 답변: "방학 중에는 시간제취업허가 보유 시 시간 제한이 완화됩니다…"
> 📄 필요서류 체크리스트 · ⏰ **비자 만료 D-43 경고** · 🔗 출처: 하이코리아 공식 안내

**Step 4 — 안전장치 시연**
> 고위험 질문: *"비자를 E-7로 바꾸려면?"* → ⚠️ "이 사안은 출입국/국제처 확인이 필요합니다" 에스컬레이션

**Step 5 — 다국어 전환**
> 같은 질문을 중국어/한국어로 전환 → 즉시 해당 언어로 응답

**발표 멘트 가이드**: Step 3에서 "**출처가 붙어 있죠? 환각이 아니라 공식 자료 기반입니다**", Step 3의 D-day에서 "**이 알림 하나가 불법체류를 막습니다**"를 강조.

## 4. 차별점 (vs 기존)

| 비교 대상 | 한계 | StayWise |
|---|---|---|
| 에브리타임·일반 학생앱 | 한국어·내국인 중심 | 다국어·유학생 전용 |
| 일반 ChatGPT | 내 체류상황 모름·최신 규정/학교절차 모름·틀리면 위험 | 개인화 + 출처기반 + 비자 특화 |
| 대구한의대 'AI기린' 등 | 단일 대학 행정챗봇 | 통합·개인화 |

## 5. 시장·임팩트

- 정부 Study Korea 30만 정책 / 지방대 유학생 유치·유지 압박
- AI in 고등교육 시장 CAGR **약 35%** (뜨거움)
- 수익화 경로: **대학 B2B** + 생활서비스 제휴(통신·은행·보험) + 정부 유치사업

## 6. 심사 예상 Q&A

| 예상 질문 | 답변 |
|---|---|
| 정확성·책임 문제는? | 공식 출처 grounding + 에스컬레이션 + 면책. "법률자문 아님, 정보·절차 안내"로 포지셔닝 |
| 기존 챗봇과 뭐가 다른가? | 개인 체류상황 기억 + 다국어 + 출처 인용 + 학교별 절차 |
| ChatGPT면 충분하지 않나? | ChatGPT는 내 비자 맥락·최신 규정·학교 절차를 모르고 환각 위험. 핵심은 개인화·출처·환각방지 |
| 규정 바뀌면? | 공식출처 모니터링 + 갱신 파이프라인 |
| 수익모델은? | 대학 국제처 B2B SaaS + 생활서비스 제휴 수수료 |

## 7. 빌드 플랜 & 역할분담 (남은 시간)

**트랙 A — 프론트엔드**
- 프로필 입력 폼 + 채팅 UI + 언어 토글 + 비자 만료 D-day 표시

**트랙 B — AI / 백엔드**
- 공식문서 ~15개 큐레이션(D-2 알바규정·비자연장·외국인등록·시간제취업허가)
- 프롬프트 설계: 출처 인용 강제 + 절차 체크리스트 + 고위험 분류 + "모르면 모른다"
- 프로필 컨텍스트 주입 + Claude API(Sonnet 4.6) 연동

**트랙 C — 데모 / 발표**
- 데모 대본·시나리오 데이터(베트남 유학생 Linh 등) 준비
- 발표자료 + Q&A 대비

**통합 우선순위 (반드시 이 순서로):**
1. 동작하는 **1개 언어** Q&A (가장 먼저 — 데모의 심장)
2. **출처 인용** 붙이기
3. **비자 만료 경고**
4. **2번째 언어** 추가
5. **고위험 에스컬레이션**

> 시간 부족 시 1~3까지만 완성해도 데모는 성립. 4·5는 여유분.
